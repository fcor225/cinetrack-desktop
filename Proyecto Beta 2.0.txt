INTRODUCCIÓN
Contextualización del problema a resolver. Describir claramente de dónde surge la necesidad de este PI y su utilidad. En caso de que se base en trabajos previos, debe aclararse cuáles son las aportaciones que ofrecería tu proyecto.
En la actualidad, el consumo de contenido audiovisual está fragmentado en múltiples plataformas de streaming (Netflix, HBO, Disney+, etc.). Los usuarios a menudo pierden el rastro de las películas que han visto, las que desean ver o las valoraciones personales que hicieron en su momento. Aunque existen plataformas web, muchas requieren conexión permanente, creación de cuentas sociales, etc. Surge así la necesidad de CineTrack Desktop, una herramienta de escritorio que centraliza la gestión de este contenido. Su utilidad principal reside en ofrecer al usuario un entorno privado, rápido y capaz de funcionar parcialmente sin conexión para organizar su vida cinematográfica, combinando la potencia de una base de datos global con la privacidad de un almacenamiento local.

Aportaciones respecto a trabajos previos:  A diferencia de los gestores de listas genéricos (Excel) o las webs sociales (Letterboxd), este Proyecto Integrado ofrece:
- Hibridación de datos: Integra información técnica automática (vía API externa) con datos subjetivos del usuario (vía Base de Datos local).
- Enfoque Educativo Intermodular: La aplicación no es solo un frontend; implementa patrones de diseńo (MVC), concurrencia para procesos en segundo plano y persistencia relacional.


OBJETIVOS
Descripción detallada de en qué consistirá el PI. En caso de que el objeto principal del PI sea el desarrollo de una aplicación, habrá que describir todo lo que queramos que haga esa aplicación. Debes detallar los objetivos, y desgranar cada uno de ellos en funcionalidades. Así mismo, deberás detallar los objetivos y funcionalidades que pretendes desarrollar para la versión 1.0 de tu proyecto. 
El objetivo principal es desarrollar una aplicación de escritorio multiplataforma utilizando el lenguaje Java. La aplicación actuará como un cliente visual que permite consultar una base de datos remota de películas (The Movie Database - TMDB) y gestionar una base de datos local embebida. 
Conexión API REST: Consumo de servicios web para obtener carteleras, búsquedas y fichas técnicas en tiempo real.
- Gestión de Usuarios (Local): Perfil único local para gestionar las preferencias.
- Sistema de Búsqueda: Filtrado por título y visualización de resultados con carga asíncrona de imágenes (pósters).
- Persistencia de Datos (CRUD): Capacidad para Crear, Leer, Actualizar y Borrar registros personales (críticas, puntuaciones, estado de "vista/pendiente").
- Interfaz Reactiva: Feedback visual al usuario durante las cargas de datos para asegurar una buena experiencia de usuario (UX).

- Objetivos para la Versión 1.0 (MVP):  Para la primera entrega funcional, el alcance se limita a:

- Buscador funcional conectado a la API de TMDB.
- Visualización de ficha técnica (Título, Sinopsis, Ańo, Puntuación Global).
- Implementación de base de datos local (SQLite) para guardar películas en "Favoritos".
- Interfaz gráfica completa desarrollada en JavaFX.




ENTORNO TECNOLÓGICO
TECNOLOGÍAS EMPLEADAS:
Enumeración de las tecnologías utilizadas (lenguajes de programación, frameworks, sistemas gestores de bases de datos, etc.) en el desarrollo del P.I.
Lenguaje de Programación: Java (JDK 17 o superior).

Interfaz Gráfica: JavaFX (con FXML para la estructura y CSS para el estilo).

Base de Datos: SQLite (Motor de base de datos relacional embebido y ligero, ideal para escritorio).

Acceso a Datos / Red:

Retrofit o HttpClient: Para peticiones HTTP a la API externa.

JDBC: Para la conexión con la base de datos local.

Gson/Jackson: Para el parseo de datos JSON.

API Externa: The Movie Database (TMDB API v3).
